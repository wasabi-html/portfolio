LP運用フローのシステム化
（更新工数削減・ヒューマンエラー防止）

キャンペーンごとに発生する価格、クーポンコード、期限などの更新作業において、複数箇所の修正漏れや、HTMLタグの誤消去によるレイアウト崩れが課題でした。
そこで、HTMLの直接編集による誤消去等のリスクを排除し、専門知識の有無に関わらず、誰もが正確かつ迅速に更新できる運用フローを設計しました。

JavaScriptで各設定値を読み込み、ページ内の該当クラスへ一括反映させる仕組みを構築しました。

その結果、1回あたりの更新作業時間を約30分から5分へと大幅に短縮し、貼り替えミスなどのヒューマンエラーを物理的に排除する運用フローを実現しました。
※割引率や価格など、半角数字での入力を徹底してもらえるようにマニュアルには記載しています。
※本デモは、仕組みを視覚化するためにアレンジした簡易版です。

サンプル3：スクリプト（JS）
以下のJSをbodyの閉じタグの直前に入れてください。 
window.addEventListener('scroll', () => {
  const banner = document.getElementById('js-floating-banner');
  const scrollValue = window.pageYOffset;
  const triggerPoint = 500; // 500pxスクロールしたら表示

  if (scrollValue > triggerPoint) {
    banner.classList.remove('is-hidden');
  } else {
    banner.classList.add('is-hidden');
  }
});

離脱を防ぐ動画ギャラリー導線設計
（モーダル再生・商品導線の最適化）

商品理解を促すための使い方動画ページを実装しました。
動画一覧UIとYouTubeモーダル再生を組み合わせ、視聴中の離脱を防ぎつつ、商品ページへの導線を自然につなげられる構成を意識しています。
※以下はモーダルの動作確認用サンプルのため、Youtube公式動画を埋め込んでいます。

担当領域・実装のポイント

  • 担当箇所：HTML / CSS / JavaScript / 動画一覧UIの構築 / YouTubeモーダル表示の実装 / スマホ閲覧を前提にしたレイアウト調整 / 既存商品ページへの導線設計
  • 工夫した点：ページ遷移によるストレスをなくすため、JavaScriptを用いてモーダル内での動画再生（YouTube iframeの制御）を実装しました。
  • レスポンシブ対応：ヘッダー画像→動画一覧→商品ページへの導線で、使い方確認から商品導線へ戻れる流れを意識した。

記事型LP実装
（読了率・CVR最適化を意識したモバイルファースト実装）

SNS広告等からの流入を受け止める「記事型LP（コラムページ）」の実装を担当しました。
長文コンテンツや多数のGIFアニメーションを含む縦長のページ構成において、ユーザーの離脱を防ぐための「読みやすい余白設計」「スマートフォンでの最適なテキスト改行・文字サイズ」を徹底。
単にデザインを再現するだけでなく、「いかにストレスなく最後まで読ませ、最終的な購入（CTA）へ繋げるか」というWebマーケティングの視点を取り入れたコーディングを行っています。

担当領域・実装のポイント

  • 担当箇所：HTML / CSS
  • 読了率の向上：スクロール時のリズムを崩さないよう、見出し、画像、テキスト間のマージンを細かく調整し、視線誘導を意識したレイアウトを構築しました。
  • パフォーマンスへの配慮：多数のGIF画像を使用するページ仕様のため、読み込み遅延による直帰を防ぐための適切な画像配置・コーディングを意識しています。
  • マーケティング視点：最終的なコンバージョン（ECショップ等への遷移）ボタンが最もクリックされやすいよう、ボタン周辺の余白や見せ方を最適化しました。
